# Tools

Tools are callable runtime capabilities. In the product, they come from more than one authoring surface, but the agent still sees one runtime-facing tool catalog.

## Important distinction

- the runtime catalog is unified
- authorship is not unified

Some tools are created directly in the tools surface. Others are mirrored in from artifacts, pipelines, or agent exports.

## First-build rule

Start with one tool that has a narrow, obvious outcome. This makes debugging much easier than starting with several broad tools.

## Runtime guardrails

- inactive tools are blocked
- production mode expects published tools
- debug mode is more permissive
- tenant visibility still applies
